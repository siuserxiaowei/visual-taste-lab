# Impeccable Reference Audit

This audit records what Visual Taste Lab can learn from the local Impeccable reference without making Impeccable part of the Visual Taste Lab brand.

## Useful Principles To Learn

- Productize design skill: Impeccable makes design guidance concrete through a named skill, command set, reference files, and a repeatable workflow. Visual Taste Lab should likewise make "taste" executable through VI audit, archetype selection, token roles, component rules, and verification steps.
- Teach against anti-patterns: its detector and README name common AI-page tells directly. Visual Taste Lab should maintain a clear "do not" list for generic AI visuals, fake proof, weak contrast, nested cards, and unstructured polish requests.
- Use systematic tokens: `tokens.css` separates fonts, spacing, widths, motion curves, and color roles. Visual Taste Lab should keep its warm paper, brick, blue, olive, gold, ink, surface, border, type, spacing, radius, and motion decisions explicit.
- Show strong visual examples: the reference uses before/after, case-study, demo, and themed microsite material to make taste visible. Visual Taste Lab should keep fictional examples, but use them to explain site-type differences rather than claim real outcomes.
- Verify visually: Impeccable emphasizes technical audits, responsive behavior, reduced motion, and rendered checks. Visual Taste Lab should keep screenshot verification in the workflow and treat mobile overflow, clipped text, and unreadable states as acceptance failures.
- Separate registers: the brand-vs-product framing in Impeccable maps well to Visual Taste Lab's company site vs product site guardrail.

## What Not To Copy

- Do not copy the Impeccable brand, name, command vocabulary, package identity, or public positioning.
- Do not copy its magenta/rose accent system, Renaissance editorial palette, `Cormorant Garamond` / `Instrument Sans` / `Space Grotesk` stack, or floating bottom navigation.
- Do not copy the Neo Mirai event aesthetic: Japanese event typography, rice/gold/night palette, vertical poem, ticket pill, speaker carousel, or conference-specific motifs.
- Do not copy its anti-pattern demo literally. Visual Taste Lab can name similar problems, but should express them through VI-first audit language and project-local examples.
- Do not reuse Impeccable's case-study framing, customer implication, downloads, npm/community copy, or external ecosystem claims.
- Do not let reference visuals override the existing Visual Taste Lab circular quadrant mark, warm paper system, brick accent, deep blue anchor, olive support color, and fictional-demo disclosure.

## Mapping To Visual Taste Lab

- Impeccable's "commands" become Visual Taste Lab's workflow stages: collect references, audit VI, classify archetype, distill design language, prototype, apply tokens/components, verify screenshots.
- Impeccable's deterministic anti-patterns become VTL acceptance checks: no generic gradient hero, no nested cards, no fake proof, no weak contrast, no clipped mobile text, no unsourced metrics.
- Impeccable's token discipline becomes VTL's design-language doc: every color, type role, radius, border, shadow, button, card, table, form, badge, and motion rule must have a job.
- Impeccable's visual confidence becomes VTL's demo strategy: show clearly fictional company/product/content examples so visitors understand method transfer without mistaking them for real clients.
